AnomalyGuard
!

AI-решение для обнаружения аномалий в потоковых данных
? О проекте
AnomalyGuard — это десктопное приложение для обнаружения аномалий в данных с использованием архитектуры нейронной сети CNN-LSTM (сверточная нейронная сеть + долгая краткосрочная память). Программа предоставляет интуитивно понятный графический интерфейс (GUI) для:

Обучения модели на исторических данных

Тестирования её производительности

Мониторинга данных в реальном времени.

Проект идеально подходит для задач проактивного обслуживания (Predictive Maintenance), мониторинга технологических процессов, обнаружения сетевых атак или анализа финансовых данных.

? Основные функции
Гибкая настройка: Возможность изменения гиперпараметров сети (количество эпох, размер пакета) через GUI.

Обучение и тестирование: Загрузка данных для обучения и оценка точности модели.

Онлайн-мониторинг: Обнаружение аномалий в потоковых данных в реальном времени.

Визуализация: Интерактивные графики для отображения потерь (loss), ошибок реконструкции (MSE) и сравнения аномалий.

? Установка и запуск
Клонируйте репозиторий:

Bash

git clone https://github.com/GusaynRizovskiy/AnomalyGuard.git
cd AnomalyGuard
Создайте и активируйте виртуальное окружение (рекомендуется):

Bash

python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для macOS и Linux:
source venv/bin/activate
Установите зависимости:

Bash

pip install -r requirements.txt
Запустите приложение:

Bash

python main.py
? Структура проекта
main.py: Главный исполняемый файл. Содержит логику GUI, управляет потоками и взаимодействует с моделью.

form_of_network.py: Макет интерфейса. Файл, сгенерированный с помощью Qt Designer.

worker.py: Рабочие потоки. Отвечает за выполнение ресурсоемких операций (обучение, тестирование) в фоновом режиме.

dark_blue_theme.qss: Файл стилей. Задает внешний вид интерфейса, включая шрифты и цветовую схему.

? Настройка
Вы можете легко адаптировать приложение под свои нужды:

Изменение стиля: Отредактируйте dark_blue_theme.qss, чтобы изменить цветовую схему, шрифты и другие визуальные элементы.

Модификация модели: Архитектура нейронной сети находится в worker.py и может быть изменена для адаптации к другим типам данных.
